### Scenario C: Data Deletion (DATA_DELETION → confirmation → mock delete)

**Step 1** — Send: **我想删除我的数据，用户名是user_bob**

Flow: IntentRouter → DATA_DELETION (critical risk) → ReactPlanner → user_data_delete tool → ToolDispatcher detects IRREVERSIBLE + userConfirmed=false → returns needsConfirmation → AgentCore saves confirmation prompt WITH pendingConfirmation metadata in metadataJson

Expected: AI asks "您确定要删除用户 user_bob 的所有数据吗？此操作不可逆。请回复"确认删除"以继续。"

**Step 2** — Send: **确认删除**

Flow: AgentCore.checkPendingConfirmation() detects confirmation keyword + reads metadata from last AI message → creates ToolCall(userConfirmed=true) → ToolDispatcher allows → UserDataDeletionService mock returns success

Expected: AI replies "您的数据删除请求已提交，预计 24 小时内处理完毕。"

### Scenario D: Transfer to Human Agent

**Step 1** — Send: **转人工**

Flow: MessageRouter detects transfer keyword → session status → HUMAN_HANDLING → HumanAgentService assigns agent_default → adds agent to GetStream channel → system message sent

Expected: "正在为您转接人工客服，请稍候..."

**Step 2** — Open new tab: **http://localhost:3000/agent**

Expected: Session appears in left sidebar with HUMAN_HANDLING status badge. Click it to see **full message history** including all AI conversations from scenarios A, B, C.

**Step 3** — Agent sends reply in the agent dashboard. It appears in real-time on the user's chat tab.

**Step 4** — User sends another message. It goes to human agent (NOT AI), visible in agent dashboard in real-time.

**Step 5** — Agent uses tool panel (帖子查询 button) to look up data and compose response.

| Issue | Diagnosis | Solution |

|---|---|---|

| FAQ search returns bad results | `SELECT count(*) FROM faq_doc WHERE embedding IS NULL` | Restart backend to re-run FaqEmbeddingInitializer |

| Messages don't appear in real-time | Check browser console for WS errors | Verify GetStream keys match between backend and frontend |

| Agent dashboard empty after transfer | `SELECT assigned_agent_id FROM session WHERE status='HUMAN_HANDLING'` | Refresh agent page, check HumanAgentService logs |
