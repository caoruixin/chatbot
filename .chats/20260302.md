评估方法论

  你的客服系统需要评估什么？

  以你的 AI 客服 chatbot 为例，评估体系分三层：

  Layer 1: 组件级评估（离线）

  ├── 意图识别准确率    → Precision / Recall / F1

  ├── 检索质量          → Context Precision / Recall（Ragas）

  ├── 回答忠实度        → Faithfulness（有没有编造信息）

  ├── 回答相关性        → Answer Relevancy（有没有答非所问）

  └── 安全性            → 幻觉率 / 毒性 / PII 泄露检测

  Layer 2: 端到端评估（离线 + 在线）

  ├── 任务完成率        → 用户问题是否被解决

  ├── 多轮对话一致性    → 上下文是否保持连贯

  ├── 工具调用正确性    → 是否调用了正确的 API/知识库

  └── 回答质量综合评分  → LLM-as-Judge 多维打分

  Layer 3: 业务指标（在线）

  ├── 用户满意度        → CSAT / 👍👎 / NPS

  ├── 人工转接率        → 越低越好

  ├── 首次解决率        → FCR (First Contact Resolution)

  ├── 平均处理时间      → AHT

  └── 成本              → 每次对话的 token 消耗 / 延迟

  5 种评估方法及选择策略

  ┌───────────────┬────────────────────────────────┬──────────────────────────────────────┬──────────┐

  │     方法      │              原理              │               适用场景               │   成本   │

  ├───────────────┼────────────────────────────────┼──────────────────────────────────────┼──────────┤

  │ 规则检查      │ 正则/关键词/格式匹配           │ 输出格式、安全词过滤、必须包含的内容 │ 几乎免费 │

  ├───────────────┼────────────────────────────────┼──────────────────────────────────────┼──────────┤

  ├───────────────┼────────────────────────────────┼──────────────────────────────────────┼──────────┤

  │ LLM-as-Judge  │ 用另一个 LLM 打分              │ 开放式回答质量评估（主力方法）       │ 中       │

  │ 人工评估      │ 人工标注队列                   │ 金标准验证集、边界案例、微调评分标准 │ 高       │

  实操建议：

  - 80% 的日常评估用 LLM-as-Judge — 成本是人工的 1/500 到 1/5000，和人类一致率约 80%

  - 15% 用规则检查 — 确定性的东西不要浪费 LLM 来判断

  - 5% 用人工评估 — 建立 30-50 条标注的验证集来校准 LLM Judge

  ---

  LLM-as-Judge 最佳实践

  这是当前最核心的评估方法，几个关键点：

  1. 评分标准要具体

     ❌ "回答质量如何？1-5 分"

     ✅ "回答是否包含事实错误？是否引用了检索到的内容？是否直接回答了用户问题？"

  2. 先推理后打分（Chain-of-Thought）

     → 让 Judge 先解释为什么，再给分数，准确率显著提升

  3. 用不同的模型做 Judge

     → 不要用同一个模型评估自己

     → 生成用 GPT-4，评估用 Claude（或反过来）

  4. 注意位置偏差

     → GPT-4 有约 40% 的位置偏差，需要做顺序随机化

  5. 用人工标注校准

     → 先让 3 个人标注 50 条，计算标注者一致性

     → 如果人之间都不一致，说明你的评分标准需要改
